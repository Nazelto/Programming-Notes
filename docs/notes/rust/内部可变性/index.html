    Rust Cell、RefCell、unsafeCell#

    使用时机#

    在结构体的状态不变，但是需要内部属性改变时。

    Cell#

    struct Temp<T>  
where
T:Copy 
{
    cache:Cell<T>
}

    Cell需要实现了Copy Trait的数据结构才能调用其get()方法。

    struct Temp {
    cache:Cell<HashMap<int,int>>
}

    而向上面HashMap没有实现Copy Trait可以使用take。

    Cell最好不要获取引用
    获取引用最好使用RefCell

    RefCell#

    struct Temp {
    cache:RefCell<HashMap<int,int>>
}

    RefCell会将编译时进行的引用检查延迟到运行时，如果运行时发生引用错误则会panic。

    由于RefCell内部通过维持一个引用计数在运行时进行引用检查，所以会导致一些性能损失。
    如果不希望有这些性能损失可以使用unsafeCell

    unsafeCell#

    struct Temp {
    cache:UnsafeCell<HashMap<int,int>>
}

    通过指针操作可变性。所有的操作都是unsafe的。
