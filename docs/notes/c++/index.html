    万能引用#

    万能引用识别条件

    1. 必须是模板参数 T&&
    2. 必须有类型推导
    template<typename T>
void func(T&& param) {
    // param 可以绑定到任何东西
}

Point p;
const Point cp;

func(p);              // T = Point&,       param 类型是 Point&
func(cp);             // T = const Point&, param 类型是 const Point&
func(Point(1, 2));    // T = Point,        param 类型是 Point&&
func(std::move(p));   // T = Point,        param 类型是 Point&&
    引用折叠规则#
    T& &   → T&      // 左值引用 + 左值引用 = 左值引用
T& &&  → T&      // 左值引用 + 右值引用 = 左值引用
T&& &  → T&      // 右值引用 + 左值引用 = 左值引用
T&& && → T&&     // 右值引用 + 右值引用 = 右值引用
    完美转发#
    template<typename T>
void wrapper(T&& param) {
    real_function(param);  // ❌ 问题：param 总是左值！
}

void real_function(Point& p) { 
    std::cout << "左值版本\n"; 
}

void real_function(Point&& p) { 
    std::cout << "右值版本\n"; 
}

Point p;
wrapper(p);              // 调用左值版本 ✓
wrapper(Point(1, 2));    // 还是调用左值版本！✗

    核心问题：即使 param 绑定到右值，param 本身作为表达式是左值！

    //std::forward 完美转发参数的值类别
template<typename T>
void wrapper(T&& param) {
    real_function(std::forward<T>(param));  // ✓ 完美转发
}

Point p;
wrapper(p);              // 转发为左值，调用 real_function(Point&)
wrapper(Point(1, 2));    // 转发为右值，调用 real_function(Point&&)

    如果形参是左值引用 如果传递了一个T类型的对象会自动为该函数创建一个左值引用。
